🚀 App Store 출시 체크리스트

앱을 처음 출시하는 사람을 위한 단계별 노트입니다. 위에서부터 천천히, 한 줄씩 짚어가 주세요.

← 솔로 빌딩 노트로 돌아가기

Phase 1

출시 전 준비

  • Apple Developer Program 가입 (연간 등록 비용 별도)
  • App Store Connect에 접속할 수 있는지 확인
  • 번들 ID 결정 — 한 번 정하면 바꾸기 어려워요
  • 앱 이름 (30자 이내) 정하기
  • 부제목 정하기
  • 카테고리 선택
  • 개인정보 처리방침 URL 준비
tip 개인정보 처리방침은 GitHub Pages에 마크다운 한 페이지만 올려도 충분합니다. 너무 어렵게 시작하지 마세요.
Phase 2

앱 빌드 & 업로드

  • Signing & Capabilities 설정 확인
  • 앱 아이콘 (1024×1024, 투명 영역 없이)
  • Launch Screen 준비
  • 버전 정하기 (예: 1.0.0)
  • 빌드 번호는 1부터 시작
  • 실기기에서 한 번 더 테스트
  • Archive → Distribute → App Store Connect
  • ‘처리 중’이 ‘준비됨’으로 바뀔 때까지 기다리기
tip Signing 문제로 한 번씩 막힙니다. 차분히 Apple Developer 계정과 Bundle ID, Profile을 다시 확인해 보세요.
Phase 3

App Store 메타데이터

  • 스크린샷 준비 (6.7" iPhone 기준, 3~5장)
  • 설명의 첫 3줄에 가장 중요한 이야기를 담기
  • 키워드 (100자 이내)
  • 지원 URL 입력
  • 연령 등급 설정
  • 가격 정책 선택 — 무료/유료
  • 출시할 국가 선택
tip 스크린샷은 Figma나 Canva 템플릿으로 출발하면 빨라요. 디자인이 자신 없어도 ‘설명문 + 화면’ 단순 조합이 의외로 잘 통합니다.
Phase 4

심사 제출

  • 업로드한 빌드 선택
  • 심사 메모 + 테스트 계정 (필요할 경우) 작성
  • ‘심사를 위해 제출’ 누르기
  • ‘심사 대기 중’ 상태 확인
tip 첫 심사는 보통 24~48시간 정도 걸립니다. 마음의 준비를 하면서, 그 시간을 두 번째 아이디어 적는 데 써 보세요.
Phase 5

리젝이 와도 괜찮습니다

  • 리젝 사유 천천히 읽기
  • 가이드라인 번호 확인하기
  • 해당 Apple 문서 찾아 읽기
  • 빌드 수정 후 다시 업로드
  • Resolution Center로 소통하기

자주 만나는 사유: 2.1 성능 · 4.3 스팸·복제 · 5.1.1 개인정보 · 3.1.1 인앱결제 · 2.3 메타데이터

note 리젝은 실패가 아닙니다. 첫 출시에서 한두 번은 거의 누구나 만납니다. 한 번 풀어보면 그 다음부터는 훨씬 수월해요.
Phase 6

출시 그 다음 날

  • ‘판매 준비됨’ 상태 확인
  • App Store에서 직접 검색해 보기
  • 가까운 3~5명에게 직접 권하기
  • 크래시 리포트 한 번 들여다보기
  • 첫 리뷰가 달리면 답글 하나 남기기
  • 축하 한마디 적어두기 — 이 순간은 한 번뿐이에요 🎉
— 출시는 끝이 아니라 시작입니다.
